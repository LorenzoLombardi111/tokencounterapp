How LLM cost calculation works

cost_usd = (input_tokens / 1,000,000) × input_rate + (output_tokens / 1,000,000) × output_rate

Both inputs and outputs are billed, with outputs almost always more expensive. A 2,000-token prompt to GPT-4o that returns 500 tokens costs (2000/1M)×$2.50 + (500/1M)×$10.00 = $0.0050 + $0.0050 = $0.01 per request. Run that 100,000 times and you've spent $1,000.

Cost optimization strategies

  1. Pick the right tier — Most workflows over-buy. GPT-4o-mini, Claude 3.5 Haiku, and Gemini Flash handle 90% of typical chat tasks at 5-20x lower cost than the flagship models.
  2. Use prompt caching — Both OpenAI and Anthropic offer 50-90% discounts on cached input tokens (above 1,024 tokens). For RAG and chatbot workflows, caching pays for itself immediately.
  3. Cap output tokens — Set max_tokens aggressively. Output is 4-5x more expensive than input on most models.
  4. Compress system prompts — A 500-token system prompt repeated 1M times = 500M tokens billed. Trim ruthlessly.
  5. Route by complexity — Use a small model to classify requests, then route only complex ones to the flagship.

Hidden costs to watch

  • Function calling overhead — Tool definitions add 100-500 tokens per request, each request.
  • Vision input — Images cost 85-1,200+ tokens depending on resolution and detail level.
  • Reasoning tokens — o1 and DeepSeek R1 emit hidden "thinking" tokens billed as output, often 1,000-10,000 per response.
  • Streaming latency — Streaming doesn't change cost but does change UX and may cause client-side over-polling.

Frequently Asked Questions

How is LLM cost calculated?

LLM cost = (input_tokens / 1,000,000) × input_rate + (output_tokens / 1,000,000) × output_rate. freetokencounter.app counts your input tokens automatically and lets you set expected output tokens. Total updates live as you edit your prompt.

Which model is cheapest for a given prompt?

It depends on the workload. For chat: Gemini 1.5 Flash ($0.075/$0.30) and GPT-4o-mini ($0.15/$0.60) are cheapest. For long-context RAG: Gemini 1.5 Flash and DeepSeek V3 lead. For reasoning: DeepSeek R1 is the cheapest reasoning-capable model. freetokencounter.app shows live cost across all 30+ models so you can compare instantly.

How accurate is the cost estimate?

Cost is calculated from token counts (within ±5% on English) multiplied by current published rate cards. It's accurate enough for budgeting and model selection, not for finance-grade accounting. Always validate against the provider's billing dashboard for production.

Does the calculator handle prompt caching discounts?

The displayed cost is the standard rate. Prompt caching (OpenAI and Anthropic both offer it) gives a 50-90% discount on the cached portion of inputs. For pipelines using caching heavily, real cost will be lower than the calculator shows.

What about output token estimation?

Output tokens are user-set in freetokencounter.app — there's no way to predict them exactly without running the model. A reasonable default is 500-1,000 for chat replies, 2,000-4,000 for long generations. Adjust to match your typical response length.

Are Llama costs really $0?

Llama models are open-weight — Meta does not run a first-party API. You pay for compute via hosts like Together, Groq, Fireworks, or Replicate. Pricing varies but typically runs $0.20-$0.90/1M input. freetokencounter.app shows tokens; check your host's rate card for cost.
